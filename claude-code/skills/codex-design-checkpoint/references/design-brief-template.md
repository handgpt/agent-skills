# Design Brief Template

Use this template when preparing a Codex design checkpoint.

```md
# Decision

One-sentence description of the design choice under consideration.

## Goal

- What outcome matters most?

## Constraints

- Technical limits
- Business limits
- Time, compatibility, security, or operational constraints

## Options Considered

1. Option A
2. Option B
3. Option C

## Current Preferred Direction

- State the choice you are leaning toward and why.
- If this direction intentionally deviates from a default best practice, state the constraint that justifies it.

## Known Risks

- Risk 1
- Risk 2

## Relevant Official Docs

- https://official-doc.example/path

## Relevant Paths

- /absolute/path/to/spec-or-doc
- /absolute/path/to/design-file
```

Keep the brief compact. Prefer summaries plus absolute local paths over whole-file pastes.
