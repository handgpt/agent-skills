# Claude Code Pitfalls

Concise project-specific pitfalls collected after successful fixes.
Read this before non-trivial implementation or review work.

## 2026-03-31 | install.sh AGENT_SKILLS_DIR resolved two levels up instead of one
- Symptom: Skills scripts fail with 'No such file or directory' because installed command paths are missing the agent-skills/ directory component
- Cause: install.sh used CLAUDE_CODE_DIR/../.. (two parent levels) to resolve AGENT_SKILLS_DIR instead of CLAUDE_CODE_DIR/.. (one level), so $AGENT_SKILLS_DIR pointed to the workspace root instead of the agent-skills repo root
- Rule: When computing parent paths in install scripts, count the directory levels carefully. Verify resolved paths against actual filesystem structure before substitution.

## 2026-03-30 | Never substitute user-specified parameter values
- Symptom: User specified gemini-3.1-pro-preview as the model but code was written with gemini-2.5-pro instead
- Cause: Agent substituted a model name from training knowledge instead of using the exact value the user provided, treating the user input as an error to correct
- Rule: When the user specifies an exact parameter value (model name, version, URL, etc), use it verbatim. Never silently replace it with what you think is correct. If you believe the value may be wrong, ask first

## 2026-03-30 | Gemini cross-directory code attribution hallucination
- Symptom: Gemini structural review reported numerous Codex references, fabricated variable names (CLAUDE_GEMINI_CLI_USER_OAUTH_TOKEN), and non-existent file names (codex-gemini-run-state.json) in claude-code/ files that actually had zero Codex references
- Cause: Gemini CLI runs with full filesystem access (--approval-mode yolo) from the workspace root. When reviewing claude-code/, it browsed the sibling codex/ directory and misattributed codex/ code patterns to claude-code/ files
- Rule: When running Gemini advisory on a subdirectory, set --project-root to that subdirectory specifically and add explicit instructions in the brief to NOT inspect sibling directories. Also verify Gemini findings against grep before acting on them
